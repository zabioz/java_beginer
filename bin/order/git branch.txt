git branch 종류

master 배포가능한 상태만을 관리

develop 다음 개발을 위한 브런치들을 병합, 배포 가능한 안정된 브런치

feature dev 브런치에서 추가로 기능을 개발하거나 버그가 생겼을때 dev에서 분기해서 작업을 진행하는 브런치

release 배포를위한 전용 브런치로 최종적인 버그수정이나 문서추가등의 직접적인 작업등을 하는 브런치, 모든작업이 끝나고 모든 기능과 추가한 기능들이 정상적으로 동작하면 master브런치에 머지
		배포가 완료되면 dev에도 머지

hotfix 출시된 master브런치에서 급하게 버그를 수정해야할때 생성하는 브런치로 수정후 곧바로 마스터브런치에 머지후 dev브런치에도 머지
